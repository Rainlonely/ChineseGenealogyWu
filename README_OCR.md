# 家谱 OCR 方案

默认方案走本地识别，避免把整本 PDF 重复送进模型。

## 现状判断

- 当前 PDF 共 212 页。
- 内部现有文本层基本只有水印 `www.ningos.com 宁格思`，不能直接用于语义分析。
- 页面主体是繁体竖排扫描件，且中段大量是家谱挂线图。

## 推荐流程

1. 先跑本地 OCR，生成带页码锚点的 Markdown 初稿。
2. 人工逐页校对，尤其重点检查挂线图页的人名、排行字、子女关系。
3. 校对完成后，再进入结构化抽取和语义分析。

## 执行命令

```bash
python3 scripts/ocr_family_genealogy.py '2卷一-212页系92—107.pdf'
```

输出文件默认是：

```text
2卷一-212页系92—107.ocr.md
```

## 可调参数

```bash
python3 scripts/ocr_family_genealogy.py '2卷一-212页系92—107.pdf' \
  --start-page 92 \
  --end-page 107 \
  --scale 3 \
  --crop-ratio 0.06 \
  --psm 5
```

## 说明

- Markdown 会按 `## 第N页` 分段，方便后续校对和定位。
- 每页会带一个平均置信度，低置信度页优先人工处理。
- 这一步只解决“先完整落字成稿”；挂线图的世系关系整理，留到人工校对后再做。

- 每页额外预留“关系描述（待校对）”区，便于把挂线图转成文字关系。
